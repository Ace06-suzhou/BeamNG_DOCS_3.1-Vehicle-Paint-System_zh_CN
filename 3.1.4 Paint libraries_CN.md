涂装库是一个包含 [涂装](/modding/vehicle/vehicle_paints/paint_definition) 和 [多涂装设置](/modding/vehicle/vehicle_paints/multi_paint_setup) 的文件，可被任意车辆引用。这样可以在同一品牌的车辆之间共享涂装、共享通用的自定义涂装，并总体上简化为车辆添加预设涂装颜色的流程。
该文件会在车辆被处理之前加载，因此在选择生成车辆时即可使用其中的内容。


### 涂装库文件

默认的涂装库文件存储在 `vehicles/common/paintLibraries/common.paintLibrary.json`。
但是任何模组作者都可以创建任意以 `.paintLibrary.json` 结尾的文件，该文件将与默认文件以相同方式运行，为所有车辆添加新的涂装和多重涂装设置。


用户自定义文件的名称和/或目录必须是唯一的，例如 `vehicles/common/myModPaints/common.paintLibrary.json` 或 `vehicles/common/paintLibraries/myMod.paintLibrary.json`。
该文件甚至可以放置在自定义车辆文件夹内，但仍可被所有车辆访问来自所有涂装库文件的全部涂装（无论其所在位置）都可以在任意车辆的主 `info.json` 文件中被引用。


涂装库文件的内容如下所示：
```js {hl_lines=[4,12]}
{
  "version": 1,
  "paints": {
    "bianco_alpino": {
      "name": "Bianco Alpino",
      "baseColor": [0.85,  0.88,  0.87,  1.2],
      "clearcoat": 0,
      "clearcoatRoughness": 0,
      "metallic": 0,
      "roughness": 0.07
    },
    "blu_acciaio": {
      "name": "Blu Acciaio",
      "baseColor": [0.25,  0.3,  0.35,  1.2],
      "clearcoat": 0,
      "clearcoatRoughness": 0,
      "metallic": 0,
      "roughness": 0.07
    },
    (...)
    }
  },
  "multiPaintSetups" : {
    "team_offroad_colors" : {
      "name": "Orange Race Colors",
      "paint1": "signal_orange",
      "paint2": "palladium_gray",
      "paint3": "black_diamond"
    },
    (...)
  }
}
```

涂装库中的涂装定义与标准涂装定义几乎完全相同。不同之处在于，涂装名称位于参数列表内部，而索引（在高亮行中显示）则作为 **Paint ID**。这样设计是为了使涂装名称在未来能够被翻译为不同语言。
在将库中的涂装设置为车辆或配置的默认涂装时，仍然使用涂装的 **name** 参数。


![](paint_name.png)
*当使用鼠标悬停在涂装上时，会显示该涂装的名称。*


### 使用库中的涂装

要在车辆上使用库中的涂装，需要在其主 `info.json` 文件中添加一个简单的段落：


```js
"libraryPaints": [
  "bianco_alpino",
  "blu_costa",
  "blu_nebbia",
  "arancione",
  "blu_marina",
  "nero_puro",
  "rosso_scarlatto",
  {"id": "argento_metallizzato",       "class": "custom"},
  {"id": "arancio_metallizzato",       "class": "custom"},
  {"id": "verde_speciale_metallizzato","class": "custom"},
  {"id": "bordeaux",                   "class": "custom"},
  {"id": "rosso_fuoco",                "class": "custom"},
  {"id": "blu_tropicale",              "class": "custom"},
  {"id": "giallo_solare",              "class": "custom"},
  {"id": "giallo_savana",              "class": "custom"},
  {"id": "verde_opale",                "class": "custom"},
  {"id": "verde_edera",                "class": "custom"},
  {"id": "carbone",                    "class": "custom"},
  {"id": "blu_brillante",              "class": "custom"},
  {"id": "azzurro",                    "class": "custom"},
  {"id": "chiaro_di_luna",             "class": "custom"}
]
```

请注意，该段落使用的是 **方括号**，而 `paintLibrary.json` 中的对应部分使用的是 **花括号**。


在添加来自库中的涂装时：
- 如果仅提供 ID，则会被视为原厂涂装。
- 如果提供包含 ID 且将 “class” 参数设置为 “custom” 的结构，则该涂装将显示在自定义涂装选项卡中。


同时使用标准涂装定义和涂装库引用的 `info.json` 文件示例如下：


```js
{
  (...)
  "defaultPaintName1": "Rosso Corallo",
  "paints": {
    "Alabastro": {
      "class": "custom",
      "baseColor": [0.84,  0.79,  0.76,  1.2],
      "clearcoat": 0,
      "clearcoatRoughness": 0,
      "metallic": 0,
      "roughness": 0.07
    },
    (...)
  },
  "libraryPaints": [
    "bianco_alpino",
    (...)
    { "id": "argento_metallizzato", "class": "custom" },
    (...)
  ]
}
```
