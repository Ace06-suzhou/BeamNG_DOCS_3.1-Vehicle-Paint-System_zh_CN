BeamNG使用 **PBR (Physically-Based Rendering)** 材质系统。就本页面而言，这意味着车辆涂装具有一些基本属性，这些属性决定了它的外观以及对光照的反应方式，包括：

- 颜色
- 金属度
- 粗糙度
- 清漆度
- 清漆粗糙度

[您可以访问此页面，查看基于物理的渲染（PBR）材质属性的概述。](/modding/materials/materials_1.5/)

位于`materials.json`文件中的材质定义，定义了每个这些属性的设置方式：
- 它可以是一个应用于整个材质的单一固定值。比如，我们可以把校车的涂装颜色直接设为黄色，用户就无法再自行调色。
但在实际应用中，漆面材质通常不会使用这种固定单一值的做法。
- 它可以是一张叠加在车辆 UV 贴图上的纹理，用来决定车辆 3D 模型不同区域分别使用哪些涂装属性。比如，一辆校车的涂装可以让车身主体呈现黄色，而某些特定部位带有黑色文字和图案/贴纸。
这种方法常见于不可调色（non-colorable）的皮肤/涂装。
- 它可以完全可调色，也可以部分可调色——也就是说，部分或全部参数会由玩家通过一个或多个调色板（共三个可用）来自由控制。比如，玩家可以自定义校车的主要车身颜色，但文字和贴花等细节始终保持黑色不变。
这种方式是漆面材质（包括基础涂装和各种皮肤）中最常见的一种，也是本页面主要讲解的内容。

### 可变色漆面材质

在`materials.json`文件内, `instanceDiffuse` p属性决定了该材质是否可调色，（true 表示可调色，false 表示不可调色）。 如果设置为true, a `color palette map` 决定了 UV 贴图的哪些部分受三个调色板中哪一个的影响。此处使用的是完整的 RGBA 颜色值：

- 红色表示将使用第一个调色板。
- 绿色表示将使用第二个调色板。
- 蓝色表示将使用第三个调色板。
- 透明（Transparent）表示不使用任何调色板（该部分不可调色）。

![](roamer_skin_twotone_uv1.color.png)
*双色调涂装设置的颜色调色板贴图示例。*

混合不同的颜色，可以实现用户在不同调色板上选择的涂装颜色所形成的渐变效果。
![](scintilla_main_skin_gradient.color.png)
*带有渐变效果的颜色调色板贴图示例。*

用户在调色板上设置的颜色随后会与材质属性进行相乘运算。例如，金属度滑块上的值会与 `metallicMap` 上的颜色值以及 `metallicFactor`中的数值相乘。在实际应用中，对于车辆涂装，乘法颜色通常设置为纯白色，数值设置为 1，以避免出现不想要的颜色混合。

通常，车辆上漆面材质的设置方式是使用一张全红色的纹理作为颜色调色板贴图，再在其上叠加一张 `opacity map`。`opacity map` 中的白色部分决定了车辆的哪些区域会被涂装覆盖，这些区域因此可以使用第一个调色板进行调色。这种设置方式会让在第二个UV贴图上引入皮肤变得非常方便。[更多信息请查看此页面。](/modding/materials/vehicle/typicalmaterials/)

### 颜色调色板贴图属性

`materials.json` 文件中包含一些额外的属性，它们会影响颜色调色板贴图的具体表现和行为：
- `colorPaletteMapUseUV`: 设置为 1 时，将使用第二个 UV 贴图。这是漆面材质的常见做法，可以让皮肤的排列和应用更加方便。
- `paletteBaseColor`:默认值为 true，如果设置为 false，则调色板的颜色设置将不会生效，材质将使用在 base color map 和/或 base color factor 中为材质设置的默认颜色值，而不是将其与调色板设置的颜色相乘。这意味着调色板将只影响 metallic、roughness、clear coat 和 clear coat roughness 的数值，而不会影响油漆的颜色。其他调色板数值也存在等效属性： `paletteClearCoat`, `paletteClearCoatRoughness`, `paletteMetallic`, `paletteRoughness`.
