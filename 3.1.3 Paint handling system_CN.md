BeamNG 在 `0.37` 更新中引入了新的车辆涂装处理系统。

车辆涂装的基本定义方式存在以下缺点：
 - 有些车辆会共用相同的涂装，例如同一品牌且生产年份相近的车型。  
采用基本系统时，这些涂装需要在每辆车的info.json文件中都重复写一遍，效率比较低。
 - 涂装名称无法被翻译成不同语言，因为涂装的名称是代码中用于查找它的键。
 - 随着时间推移，不同车辆的涂装列表之间会积累了一些不一致的情况。  
例如，可能会出现两辆车上同名涂装但数值不同的情况，或者反过来，数值相同但名称不同的情况。
 - 基本系统维护起来很麻烦，因为只要某个涂装设置需要改动，就得在每一辆使用这个涂装的车辆的文件里都做一遍相同的修改。
 - 有时单一涂装并不能完全覆盖车辆的外观。比如有些车是双色调设计，或者涂装直接用了三种颜色。  
这样一来，如果生成车辆时只选了第一个调色板，就很容易出现颜色搭配不合适的情况。
 - 不管是哪种配置，甚至是自定义涂装或多色皮肤里的局部细节，所有涂装都必须记录在车辆的主info.json文件里，这样其他配置也能访问到。结果就是在生成车辆时，涂装选择列表会变得异常庞大，不太现实。
 - 交通车辆在生成或传送时，涂装颜色会随机挑选，这使得整个车辆池在任何时候的涂装颜色分布看起来都不够真实。为了修正这个问题，需要复制某些涂装，让它们有更高的随机选中几率。

新的涂装系统利用了一些新概念来修复以上所有问题。  
这些内容将在后续页面中详细说明：
  - [涂装库](/modding/vehicle/vehicle_paints/paint_library): 一个存放涂装和多涂装设置的文件，所有车辆都可以共用它，避免了为每辆车重复定义涂装的麻烦。
  - [多涂装设置](/modding/vehicle/vehicle_paints/multi_paint_setup): 一套三种涂装的组合，一次性全部应用到车辆。它与普通涂装一同存在于两个涂装选择菜单中。
  - 涂装分类: 涂装分为原厂和自定义两类。只有原厂涂装会在车辆选择器中显示。原厂和自定义涂装，以及用户自制的涂装，会在车辆配置器的涂装菜单中分不同标签页显示。
  - 涂装层级结构：Library > Model > Config。
在 Library 中定义的涂装和多重涂装设置可在车辆 Model 以及车辆 Config 中使用。
在车辆 Model 中定义的涂装可在该 Model 的任意 Config 中使用。
在车辆 Config 中定义的涂装仅可被该 Config 使用。
  - [随机涂装分布](/modding/vehicle/vehicle_paints/paint_distribution)：可以为涂装分配一个权重值，用于影响其在交通车辆中被选中的概率，从而实现更真实的颜色分布。
