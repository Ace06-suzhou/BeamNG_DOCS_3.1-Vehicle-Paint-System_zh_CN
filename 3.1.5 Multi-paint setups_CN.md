多涂装设置由最多 3 个 [涂装](/modding/vehicle/vehicle_paints/paint_definition) 条目以及一个设置名称组成。每个涂装都会应用到其对应的 [调色板](/modding/vehicle/vehicle_paints/paint_materials_intro)，从而在生成车辆时能够一次性设置所有可用的涂装颜色。

![](multipaint_veh.png)
*在生成车辆时选择多涂装设置。*

多涂装设置会在车辆配置器的涂装菜单中以独立选项卡的形式显示。
![](multipaint_part.png)
*车辆配置器中的多涂装设置。*

### 定义多涂装设置


多涂装设置可以定义在 [涂装库](/modding/vehicle/vehicle_paints/paint_library) 中，或定义在车辆主 `info.json` 文件中。在这两种情况下，其定义形式相同：

```js
"multiPaintSetups" : {
  "team_offroad_colors" : {
    "name": "Orange Race Colors",
    "paint1": "signal_orange",
    "paint2": "palladium_gray",
    "paint3": "black_diamond"
  },
  "rennspecht_green_colors" : {
    "name": "Rennspecht Green Colors",
    "paint1": "rennspecht_green",
    "paint2": "palladium_gray"
  },
  (...)
}
```

键 `"paint1"`、`"paint2"` 和 `"paint3"` 分别对应将应用于第 1、第 2 和第 3 个调色板的 **Paint IDs**。
如果该设置定义在 [涂装库](/modding/vehicle/vehicle_paints/paint_library) 文件中，则所引用的涂装也必须是库涂装。若定义在 `info.json` 文件中，则也可以使用标准涂装，此时使用涂装名称而非 ID。
如果未定义 `paint2` 或 `paint3`，则对应的调色板将默认使用 `paint1`。


车辆的 `info.json` 文件可以同时使用其自身定义的多涂装设置以及来自涂装库的通用多涂装设置。
```js
"multiPaintSetups": [
  "rennspecht_green_colors",
  "rennspecht_black_colors",
  "rennspecht_blue_colors",
  {
    "name": "Battlehawk Purple",
    "paint1": "Extra Purple",
    "paint2": "Satin Black",
    "paint3": "Satin Gray"
  },
  {
    "name": "Battlehawk Green",
    "paint1": "Goblin Green",
    "paint2": "Satin Black",
    "paint3": "Satin Gray"
  },
],
```
在这两种情况下，所有来自涂装库的涂装颜色都必须事先在 `libraryPaints` 段落中声明。可以将其分配为 [custom class](/modding/vehicle/vehicle_paints/paint_definition)，以避免涂装选择器中过于拥挤。


多涂装设置也可以仅包含 1 个涂装，并使用该涂装的名称，从而在界面上伪装成一个普通的涂装选项。
这对于默认配备双色涂装设计的车辆来说非常有用，可用于添加一个选择单色涂装的选项。


### 将多涂装设置用作默认值

无论是在车辆主 `info.json` 文件中，还是在任意配置对应的 `info_[config name].json` 文件中，多涂装设置都会替代旧的"defaultPaintName1/2/3"值。
旧方法仍然可用于向后兼容，但 `defaultMultiPaintSetup` 具有更高优先级。
它可以通过以下方式之一使用：


- 通过提供在 `info.json` 中定义或从库中导入的多涂装设置名称。例如：defaultMultiPaintSetup":"Champagne"。
在这种情况下，通过车辆选择器生成车辆时，该多涂装设置将被高亮显示。


![](multipaint_selection.png)
*该配置的默认多重涂装设置会被高亮显示。*

- 通过为此配置定义专属的多涂装设置。例如：
```js
"defaultMultiPaintSetup":
{
  "name": "Turbo Race",
  "paint1": "Royal Blue",
  "paint2": "Solar Yellow",
  "paint3": "Solar Yellow"
}
```
在这种情况下，该设置仅会在选中此特定配置时出现在多涂装列表中。



![](multipaint_selection2.png)
*在选择使用该设置的自定义配置时，会自动添加默认的多涂装设置。*


建议模组作者完全使用多涂装设置来替代 `defaultPaintName` 语法的用法。
